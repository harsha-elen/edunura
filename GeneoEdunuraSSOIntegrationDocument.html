Geneo × Edunura - SSO Integration

Document Version: 1.0

Date: 2026-03-28

Prepared by: Geneo Engineering Team

Shared with: Edunura Engineering Team

1. Overview

This document describes the technical requirements for integrating Edunura's identity system with the Geneo learning platform via Single Sign-On (SSO).

  • Goal: Allow Edunura students to log in to Geneo directly using their existing Edunura credentials—no separate Geneo account creation needed.
  • Integration Model: Geneo calls Edunura's APIs. Edunura does not call Geneo APIs directly. Edunura's frontend/app initiates the login and passes the token to Geneo.

2. High-Level Flow

[Edunura App / Browser]

1. User is logged in to Edunura (has JWT token + uniqueId/guid).

2. User clicks "Open in Geneo" button in the Edunura app.

https://learn-stage.geneo.in/sso-redirect/edunura?uId={uniqueId}&token={jwt}

[Geneo Web App / Browser]

4. Geneo frontend reads guid + token from URL params.

5. Geneo frontend calls Geneo backend SSO API:

POST /validateUserSsoLoginV3 { token, uniqueId: uid, role, organizationCode: ‘EDUNURA001’ }

[Geneo Backend]

6. Geneo backend calls Edunura User Info API:

POST {EDUNURA_USER_INFO_ENDPOINT}

Authorization: Bearer {token}

Body: { uniqueId }

7. Geneo validates the user and creates/updates their profile.

8. Returns Geneo user profile + session cookie to the frontend.

[Geneo Web App / Browser]

9. User is now logged in to Geneo.

Key Point: Edunura only needs to perform a browser redirect (Step 3). Edunura does not call any Geneo API directly. Geneo's own frontend handles the rest.

3. One-Time Setup (Geneo Side)

Before any login can happen, Geneo needs to be configured with Edunura's organization and school data. This is done once by the Geneo team using Edunura's Organization Info API (see Section 6.1).

Geneo will call:

POST /createSsoOrganizationV2
{
 "ssoProvider":         "EDUNURA",
 "organization":        "edunura",
 "organizationCode":    "EDUNURA-001",
 "currentAcademicYear": "AY_2025_2026"
}

This creates all school, class, and section mappings inside Geneo by calling Edunura apis, next section mentions the apis.

4. What Edunura Needs to Implement on Their Side

4.1 "Open in Geneo" Button / Redirect

Edunura's only frontend responsibility is to add a button or link that redirects the user's browser to Geneo's SSO landing page, appending the user's guid (unique ID) and JWT token as query parameters.

Redirect URL format:

https://app.geneo.in/sso-redirect/edunura?guid={uniqueId}&token={jwt}

Example:

https://app.geneo.in/sso-redirect/edunura?guid=058372de-9ee3-47ee-af79-bc6042912b6c&token=eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

Parameters:

Parameter

Description

guid

The logged-in user's unique identifier in Edunura's system (same as sub in the JWT payload).

token

The JWT token issued by Edunura SSO after the user logs in (without the Bearer prefix—just the raw token).

Geneo's web app receives this redirect, extracts the guid and token, determines the user's role, and handles the rest of the login flow internally. Edunura does not need to call any Geneo API.

5. Geneo's Internal SSO API (For Reference Only)

This section documents the API that Geneo's own frontend calls after receiving the redirect. Edunura does not call this API—it is included here for transparency and debugging purposes only.

5.1 Validate SSO Login (Called by Geneo frontend)

  • Endpoint: POST https://{geneo-api-host}/ums/geneo.ums.sso.login.apis.UmsSsoLoginApiServiceV1/validateUserSsoLoginV3
  • Content-Type: application/json

Request Body:

Field

Type

Description

requestOrigin

string

Always "EDUNURA"

uniqueId

string

Value from guid query param

token

string

Bearer {token} — value from token query param prefixed with Bearer

role

string

"PROFILE_ROLE_STUDENT" or "PROFILE_ROLE_TEACHER"

organizationCode

string

"EDUNURA-001"

Student Example Request:

{
 "requestOrigin": "EDUNURA",

 "role": "PROFILE_ROLE_STUDENT",

6. APIs Edunura Must Provide to Geneo

Geneo needs two HTTP endpoints from Edunura. Both must be accessible by Geneo's backend servers. Please share the base URLs and authentication credentials separately over a secure channel.
